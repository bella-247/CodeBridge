LeetCode → GitHub — Options & Instructions

Quick start
  • Open a LeetCode problem page and click the extension icon.
  • Click "Sign in with GitHub" and follow the displayed instructions. You will see a short code and a verification URL in the popup — copy or use the provided "Open page" button to navigate to GitHub's device verification page, then paste the code to authorize.
  • If you check "Remember me" before signing in, the extension saves your OAuth token locally so you won't need to sign in again on this browser.
  • After signing in, detect the problem and click "Save to GitHub". The extension will automatically create the repository (if missing) and upload a folder containing your solution and README.
Privacy & storage
  • No Personal Access Tokens (PATs) are requested or stored by default.
  • OAuth tokens are stored only when you explicitly check "Remember me" in the popup during sign-in.
  • To sign out and remove any stored token use the "Sign out" button in the popup, or clear the extension storage from this page.
Defaults
Auto-save will attempt to push detected solutions automatically when LeetCode signals a successful submission; enable with care.
